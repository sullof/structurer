Structurer

Shape stories with visual narrative planning maps

A private workspace for story structure

Structurer helps writers build stories with visual planning maps, color-coded notes, characters, and narrative phases across built-in and custom structures.

Privacy first

Everything you write stays in your browser local storage. No server-side database is used, no user accounts are created, and no cookies are required for app data. You can review details in Privacy Policy and Terms (footer links).

Portable stories

Export any story as JSON and import it in another browser or computer. Your format is consistent across instances of Structurer.

You can also import community extension packs (custom structures) from the dashboard ⋯ Actions → Structure → Import/merge custom structures, then pick Import from a file or Paste custom structures JSON.

    Help

    Structurer FAQ

    Click a question to expand the answer.

    What is a story structure?

    A story structure is an organizational framework that helps shape a narrative arc: setup, rising tension, turning points, climax, and resolution. It does not replace creativity, but gives you a map to manage pacing and progression.

    If you want background reading, start with Narrative structure (Wikipedia).

    What is a story?

    A story is your main planning workspace. Every story uses one narrative structure (for example Hero's Journey, Story Circle, or a custom structure) and renders each phase as a column.

    You can think of a story as a visual planning canvas: you place notes inside phases and refine the story over time. To reorder phase columns by dragging, the story must use an altered structure (Story actions → Edit the structure creates a story-specific copy). A story also stores its own phase order, notes, and update history used for merge operations during import.

    What are series and when should I use them?

    Series let you organize multiple stories into a sequence. This is useful for multi-arc projects, series planning, or alternate structure explorations of the same story.

    Use Create series in the dashboard to name a series and pick stories with checkboxes. To add an existing story to a series later, open that story’s actions and choose Add story to series (available once you have at least one series), then pick the series from the list. Manage order, rename the series, and membership from Edit series in series actions on the dashboard.

    How do I rename a story or a series?

    Stories: double-click the story title in the dashboard list, on the story board header, or on a story title inside a series view. The title becomes an editable field: press Enter to save, Escape to cancel, or click outside to save.

    Series: open Edit series from series actions (dashboard or series page) and change the Series name field at the top of the dialog.

    Can I modify the demo stories to learn the app?

    Yes. Feel free to edit and experiment with demo stories (move notes, change phases, and tweak everything you want).

    If you ever want to restore the demos, go to the dashboard and run Reset demos. Your personal stories and settings will be kept.

    Hide demos can also read Hide demos and AI analyses when you have stories imported from the Build AI import prompt flow (JSON with aiAnalysisImport: true). That toggle hides built-in demos and those AI-tagged boards together.

    Which story structures are pre-built in Structurer?

    Structurer currently includes these pre-built structures:

    • Hero's Journey
    • Hero with a Thousand Faces
    • Three-Act Structure
    • Save the Cat
    • Story Circle
    • 7-Point Story Structure
    • Romancing the Beat
    • MICE Quotient
    What are phases and notes?

    Phases are the structure steps (columns). Notes are cards inside those columns. Notes can represent plot points, themes, characters, or custom note types.

    Drag notes with the handle (⋮⋮) to reorder them inside a phase or move them across phases. When the story uses an altered structure, a second handle beside the phase title lets you reorder columns. You can collapse notes for compact reading and open them in edit mode when needed.

    How do I create a note?

    In the story editor, click + in the phase header. Then select the note type from the menu. Character notes open from the character submenu so you can choose an archetype immediately.

    New notes are saved locally and can be moved, collapsed, edited, and resized at any time.

    My note is long — can I collapse it?

    Yes. Double-click the note header. The card collapses to a one-line preview so you can scan the story quickly; double-click again to expand.

    How do I create custom note types and custom archetypes?

    Open the story board, then Actions → Note types & archetypes:

    • Add custom note type to create a new note category and choose its color.
    • Add custom archetype to add a new character archetype.

    These custom definitions are saved in your browser and become reusable across stories.

    I want to use a structure that is not listed. What should I do?

    Go to dashboard and open Add a structure not listed. For each phase, enter a name and optionally a short description (shown as help when editing a story), save, and the new structure will appear in the story creation selector.

    For example, if you want to follow a structure like Freytag's Pyramid (which is not included by default), you can add its phases manually and save it as your own reusable structure.

    You can also import ready-made custom structure packs from dashboard ⋯ Actions → Structure → Import/merge custom structures.

    I don't speak English. Where can I find structures in my language?

    The built-in structure presets in Structurer are in English. For frameworks labeled in other languages (or regional narrative patterns), use community extensions: they are custom structure packs you import into the app so they appear when you create a new story.

    For what extensions are, where to download JSON, and how to import (file or paste), see the next question: What are extensions and how do I use them?

    What are extensions and how do I use them?

    Extensions are curated custom structure packs distributed as JSON files. They let you add new narrative frameworks (including non-English variants) without changing the built-in defaults.

    To use one: download a JSON file from the extensions catalog, open dashboard ⋯ Actions → Structure → Import/merge custom structures, then choose Import from a file or Paste custom structures JSON.

    See the catalog in EXTENSIONS.md and contribution rules in CONTRIBUTING_EXTENSIONS.md.

    The work I want to analyze is not in the demos. How do I map it to a structure?

    Demos are only examples. On the dashboard, open the last card under Stories (“Want analyses that aren’t in the demos?”) or use Build AI import prompt below: you get a ready-made prompt for an AI assistant (instructions stay in English; you can choose the language for the analysis text in the notes). Full steps for importing the JSON are on that page after you generate the prompt.

    AI output can contain mistakes; review notes after import. Use imports for personal study only and respect the work’s copyright.

    How do export, import, and merge work?

    Export a story from dashboard story actions. Import from ⋯ Actions → Story → Import/merge a story, then choose Import from a file or Paste story JSON in the dialog.

    • If the imported story UID is new, Structurer creates a new story.
    • If the imported story UID already exists, Structurer merges notes by note UID.
    • When both sides changed, the most recent update wins.

    Recommended multi-device workflow: do one initial export/import to align IDs between the two copies. After that, repeat export/import for incremental merges.

    If phase order differs between the two versions, merge is blocked and Structurer shows a side-by-side conflict modal so you can align the story manually and retry.

    I created a story or a custom structure. Can I share it just by sending a link?

    No. Structurer keeps your work in this browser’s storage only. The hosted app has no server-side copy of your boards and no per-user URLs that carry your data. If you send someone a link (even a story URL with a slug), they only open their local app — they will not see your story or structures.

    To share a story, export it from Story actions (⋯ on the board) and send the JSON file or text. The recipient imports it from the dashboard ⋯ Actions → Story → Import/merge a story (Import from a file or Paste story JSON).

    To share a structure template (phases only, not note content), open it from Available structures on the dashboard, then use Export in the preview and send that JSON. The recipient uses ⋯ Actions → Structure → Import/merge custom structures. The file uses exportType: "structurer.structure" (legacy extension packs still use structurer.custom-structures). Built-in templates keep their real id (e.g. hero_journey). On import, any entry whose id is already a built-in framework in the recipient’s app is skipped so you do not get a duplicate; if that id is not built-in there, it is merged into custom structures like any other template.

    How can I add phase comments without cluttering the story board?

    Open a story, then click the magnifier icon on a phase header. Structurer opens a dedicated phase details page where you can keep writing comments without overcrowding the main board.

    On desktop, notes for that phase are shown on the left (read-only for context) while comments are on the right. This keeps your story board clean while preserving all deeper notes.

    You can add, edit, and delete as many comments as you need. Back in the story board, the phase shows a comment counter so you always know where extra context exists.

    I published my novel. How do I delete everything from Structurer?

    You can delete your stories and series one by one from the dashboard (use story/series actions).

    If you want to do everything quickly, go to the dashboard and use Factory reset app data. Be absolutely sure before confirming: this permanently deletes all your local data from the browser storage (stories, series, custom structures/archetypes/note types, and app settings). There is no way to recover it.

    If you only want to restore the demo stories, use Reset demos instead. And if you want to keep your current work, export stories first.

    5. Build AI import prompt

    Build AI import prompt

    Pick a structure template and the work’s title. Structurer builds one block of text (in English) to paste into ChatGPT, Claude, or another LLM—ask it to reply with only valid JSON in Structurer’s import shape. Choose the language for the analysis text inside the notes separately so non-English writers get readable output without translating the instructions themselves.

    After the AI returns JSON

    1. Copy the model’s answer as plain JSON (strip markdown code fences if it wrapped the payload).
    2. The built-in prompt asks for "schemaVersion": 3, a canonical "structureId" (app framework id), a human-readable "structure" name, and root "aiAnalysisImport": true. Keep those fields so Structurer loads the correct columns and tags the board as AI analysis (badge + Hide demos and AI analyses). If the model omits aiAnalysisImport, the import is treated like a normal story.
    3. On the dashboard, open ⋯ Actions → Story → Import/merge a story, then Paste story JSON, and paste the JSON into Structurer.
    4. If the imported story UID is new, Structurer creates a board. If that UID already exists, Structurer merges notes by note UID (see the import FAQ for conflicts).

    AI output can contain mistakes; review notes after import. Use imports for personal study only and respect the work’s copyright.

    Privacy Policy

    Privacy Policy

    Simple, minimal, and personal.

    Structurer is a personal app. It runs entirely in your browser for planning and note-taking.

    All your data is saved locally in your browser storage. No server-side database is used, and no user accounts are created.

    The app is served by a simple web server whose only role is to deliver the app files. There is no tracking, no cookies, and nothing is sent to any third party.
